Tema 1

Validacion, Verificacion y Testing

Validacion y verificacion son dos controles distintos pero conectados. Validar confirma que la solucion responde a la necesidad real del usuario. Verificar confirma que lo construido respeta la especificacion, historia o criterio definido.

Validacion

Pregunta clave: estamos construyendo el sistema correcto?

Verificacion

Pregunta clave: estamos construyendo correctamente el sistema?

Conceptos clave

  • Testing es verificacion dinamica porque ejecuta el software.
  • Las pruebas muestran defectos, pero no demuestran ausencia total de errores.
  • Validar temprano evita construir sobre una necesidad mal entendida.

Ejemplo

Si el cliente necesita recuperar contrasena por email, validar es confirmar que ese flujo resuelve su necesidad. Verificar es probar que el sistema envie el email correcto cuando se usa la opcion definida.

Resumen final: validar mira si hacemos lo correcto; verificar mira si lo hacemos bien; testing ejecuta el sistema para obtener evidencia.
Tema 2

Plan de pruebas

El plan de pruebas define objetivos, alcance y lineamientos para guiar el trabajo de testing. Sirve para ordenar que se prueba, quien participa, como se prueba, con que criterios y bajo que riesgos.

Conceptos clave

Scope People Approach Criteria Environment Deliverables Incidents Risk Task

Ejemplo

Para probar un ABM de usuarios, el alcance puede incluir alta, baja, modificacion y busqueda. Puede dejar fuera pruebas de usabilidad si las hace UX. Los criterios pueden exigir cero defectos bloqueantes antes de liberar.

Resumen final: un plan de pruebas responde que, quien, como, donde, cuando, con que criterios y que riesgos existen.
Tema 3

Casos de prueba

Un escenario es el titulo significativo de lo que se quiere probar. Un caso de prueba es el paso a paso ejecutable con datos, precondiciones y resultado esperado.

Conceptos clave

  • Escenario: que se quiere probar a grandes rasgos.
  • Caso de prueba: como se prueba paso a paso.
  • Debe existir trazabilidad entre historia, criterio, escenario, caso y defecto.
  • Hay casos positivos y negativos.

Ejemplo

Escenario: inicio de sesion exitoso con credenciales validas.

Caso: abrir login, ingresar email valido, ingresar contrasena correcta, presionar ingresar y verificar acceso a Home.

Resumen final: escenario dice que probar; caso de prueba dice como probarlo y que resultado esperar.
